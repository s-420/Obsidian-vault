
## 简历上传功能 - 文件去重实现总结

### 一、去重核心原理

```
文件去重 = 计算文件内容哈希 + 数据库查询
```

**核心思想**：相同内容的文件，其 SHA-256 哈希值也相同。

---

### 二、去重流程图

```
用户上传简历
    ↓
┌─────────────────────────────────────────┐
│ 1. 计算文件哈希值                        │

└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐

    │         更新访问次数
    │         跳过上传和解析
    │
    └─ 不存在 → 继续正常上传流程
```

#### 步骤 1：计算文件哈希

```java
public String calculateHash(MultipartFile file) {
    try {
        return calculateHash(file.getBytes());
    } catch (IOException e) {
        throw new BusinessException(ErrorCode.INTERNAL_ERROR, "计算文件哈希失败");
    }
}

public String calculateHash(byte[] data) {
    try {
        MessageDigest digest = MessageDigest.getInstance("SHA-256");
        byte[] hashBytes = digest.digest(data);
        return bytesToHex(hashBytes);  // 转换为十六进制字符串
    } catch (NoSuchAlgorithmException e) {
        throw new BusinessException(ErrorCode.INTERNAL_ERROR, "计算文件哈希失败");
    }
}
```

**哈希算法**：SHA-256

**输出示例**：
```

```

#### 步骤 2：检查文件是否已存在

**位置**：[ResumePersistenceService.findExistingResume()](file:///D:\Code\JavaProjects\interview-guide\app\src\main\java\interview\guide\modules\resume\service\ResumePersistenceService.java#L25)

```java
public Optional<ResumeEntity> findExistingResume(MultipartFile file) {
    try {
        // 1. 计算文件哈希
        String fileHash = fileHashService.calculateHash(file);
        
        // 2. 查询数据库
        Optional<ResumeEntity> existing = resumeRepository.findByFileHash(fileHash);
        
        // 3. 如果存在，更新访问次数
        if (existing.isPresent()) {
            log.info("检测到重复简历: hash={}", fileHash);
            ResumeEntity resume = existing.get();
            resume.incrementAccessCount();  // 访问次数 +1
            resumeRepository.save(resume);
        }
        
        return existing;
    } catch (Exception e) {
        log.error("检查简历重复时出错: {}", e.getMessage());
        return Optional.empty();
    }
}
```

```java
public Map<String, Object> uploadAndAnalyze(MultipartFile file) {
    // 1. 验证文件
    fileValidationService.validateFile(file, MAX_FILE_SIZE, "简历");
    
    // 2. 验证文件类型
    String contentType = parseService.detectContentType(file);
    validateContentType(contentType);
    
    // 3. 检查简历是否已存在（去重）⭐
    Optional<ResumeEntity> existingResume = persistenceService.findExistingResume(file);
    if (existingResume.isPresent()) {
        log.info("简历上传处理完成（重复）: {} - 耗时: {}ms",
            fileName, System.currentTimeMillis() - startTime);
        return handleDuplicateResume(existingResume.get());  // 返回历史结果
    }
    
    // 4-8. 如果不存在，继续正常上传流程
    // 解析文本 → 保存文件 → 保存数据库 → 发送分析任务 → 返回结果
}
```

#### 步骤 4：处理重复简历

**位置**：[ResumeUploadService.handleDuplicateResume()](file:///D:\Code\JavaProjects\interview-guide\app\src\main\java\interview\guide\modules\resume\service\ResumeUploadService.java#L135)

```java
private Map<String, Object> handleDuplicateResume(ResumeEntity resume) {
    log.info("检测到重复简历，返回历史分析结果: resumeId={}", resume.getId());

    // 获取历史分析结果
    Optional<ResumeAnalysisResponse> analysisOpt = persistenceService.getLatestAnalysisAsDTO(resume.getId());

    // 已有分析结果，直接返回
    return analysisOpt.map(resumeAnalysisResponse -> Map.of(
        "analysis", resumeAnalysisResponse,  // 完整的分析结果
        "storage", Map.of(
            "fileKey", resume.getStorageKey(),
            "fileUrl", resume.getStorageUrl(),
            "resumeId", resume.getId()
        ),
        "duplicate", true  // 标记为重复
    )).orElseGet(() -> Map.of(
        // 没有分析结果（可能之前分析失败），返回当前状态
        "resume", Map.of(
            "id", resume.getId(),
            "filename", resume.getOriginalFilename(),
            "analyzeStatus", resume.getAnalyzeStatus() != null 
                ? resume.getAnalyzeStatus().name() 
                : AsyncTaskStatus.PENDING.name()
        ),
        "storage", Map.of(
            "fileKey", resume.getStorageKey(),
            "fileUrl", resume.getStorageUrl(),
            "resumeId", resume.getId()
        ),
        "duplicate", true
    ));
}
```

### 五、去重优势

| 优势 | 说明 |
|------|------|
| **节省存储空间** | 相同文件只存储一次 |
| **节省 AI 成本** | 相同简历只分析一次 |
| **提升响应速度** | 重复文件直接返回历史结果 |
| **数据一致性** | 确保同一简历的分析结果一致 |
| **访问统计** | 通过 `access_count` 统计热门简历 |

### 八、注意事项

⚠️ **哈希算法选择**：
- 使用 **SHA-256**（安全性高，碰撞概率极低）
- 不使用 MD5 或 SHA-1（已被证明不安全）

⚠️ **文件大小限制**：
- 大文件计算哈希可能耗时
- 当前实现使用 `file.getBytes()`，适合小文件
- 超大文件建议使用流式计算（`calculateHash(InputStream)`）

⚠️ **哈希冲突**：
- SHA-256 理论上存在冲突，但概率极低
- 实际应用中可以忽略

### 九、总结

```
简历去重功能实现：

1. 计算文件 SHA-256 哈希值
   ↓
2. 查询数据库是否存在该哈希
   ↓
3. 存在 → 返回历史结果，更新访问次数
   不存在 → 继续正常上传流程
   ↓
4. 保存时记录哈希值到数据库
```

**核心优势**：节省存储、节省成本、提升速度！
