# AI Interview Platform 架构体系

## 核心思想：**依赖注入 + 组合 + 异步消息**

```
┌─────────────────────────────────────────────────────────────┐
│                      Controller 层                          │
│  (路由 + 参数校验，委托给 Service，不写业务逻辑)              │
└─────────────────────────┬───────────────────────────────────┘
                          │ 注入多个专门 Service
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      Service 层                              │
│  按职责拆分，每个 Service 只做一件事                          │
│  ┌─────────────────┐ ┌─────────────────┐ ┌─────────────┐  │
│  │ResumeUploadService│ │ResumeParseService│ │ResumeGrading│  │
│  └────────┬────────┘ └────────┬────────┘ └──────┬──────┘  │
│           │ 依赖组合           │                  │          │
└───────────┼───────────────────┼──────────────────┼──────────┘
            ▼                   ▼                  ▼
┌─────────────────────────────────────────────────────────────┐
│                   Infrastructure 层                         │
│  (技术能力复用，不写业务逻辑)                                │
│  ┌──────────────┐ ┌──────────────┐ ┌────────────────────┐  │
│  │FileStorage   │ │RedisService  │ │PdfExportService   │  │
│  │Service       │ │              │ │                    │  │
│  └──────────────┘ └──────────────┘ └────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 与传统"苍穹外卖"的区别

| 对比项        | 传统项目 (如苍穹外卖)               | 本项目                                           |
| ---------- | -------------------------- | --------------------------------------------- |
| Service 设计 | 一个大 Service 类包揽所有          | 多个小 Service 按职责拆分                             |
| 依赖方式       | Controller → Service → Dao | **Controller → 多个专门Service → Infrastructure** |
| 外部调用       | Service 直接调 HTTP/DB        | Service 注入 Infrastructure 服务                  |
| 异步处理       | 同步或简单线程池                   | **Redis Stream + Producer/Consumer 模板**       |
| 组合模式       | 直接调用                       | **依赖注入 + 组合**                                 |

## 关键设计模式

### 1. 模板方法模式 (AbstractStreamProducer/Consumer)

```java
// 固定骨架，子类只需实现抽象方法
public abstract class AbstractStreamProducer<T> {
    protected void sendTask(T payload) {
        // 固定流程：校验 → 发送 → 记录 → 失败处理
    }
    protected abstract Map<String, String> buildMessage(T payload);
}
```

### 2. 组合模式 (Service 注入 Service)

```java
// ResumeUploadService 组合了多个专门服务
private final ResumeParseService parseService;
private final FileStorageService storageService;
private final ResumePersistenceService persistenceService;
```

### 3. 外部服务封装 (Infrastructure 层)

```java
// Service 不直接调 S3/Redis，委托给 Infrastructure
private final FileStorageService storageService;  // 封装了 AWS S3 SDK
private final RedisService redisService;          // 封装了 Redisson
```

## 总结

这个项目采用的是 **DDD-lite + 依赖注入 + 异步消息** 的现代化 Spring Boot 架构：

1. **Controller** 职责极简，只做路由
2. **Service** 按职责拆分，组合使用
3. **Infrastructure** 封装技术能力，供 Service 调用
4. **异步任务** 通过 Redis Stream 解耦
