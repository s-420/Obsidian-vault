## 一、今日学习内容

### 1. 解码与采样参数核心概念

**前置核心认知**：
- **解码（Decoding）**：把大模型输出的原始分数（logits），转换成人类可读文本的全过程
- **采样（Sampling）**：按设定规则从词表概率分布选出下一个Token
- 核心本质：在「输出稳定性」和「输出多样性」之间做权衡

**大模型单步生成流程**：
```
输入计算（Prompt + Context）→ 概率转换（softmax）→ 采样选Token → 循环生成
```

### 2. 核心采样参数详解

**Temperature（温度系数）**：
| 温度区间 | 效果 | 适用场景 |
|---------|------|---------|
| T=0 | 贪婪解码，理论上完全确定 | 极致稳定的结构化输出 |
| 0<T≤0.3 | 低温区，高度收敛稳定 | JSON/XML结构化输出 |
| 0.3<T≤0.8 | 平衡稳定性和多样性 | 数据分析、内容总结 |
| T>1.2 | 高温区，随机性极强 | 极端创意创作 |

**Top-k 与 Top-p（核采样）**：
- Top-k：只保留概率最高的前k个Token
- Top-p：累加概率达到p值的最小Token集合作为候选池
- 黄金组合：**低温（0.1~0.3）+ Top-p=0.9**

### 3. 生成控制参数

**Max Tokens & Stop Sequences**：
- Max Tokens：单次最多生成Token数（硬上限）
- Stop Sequences：遇到指定字符串立即停止（软终止）
- 重要认知：上下文窗口 ≠ 最大生成长度

**Repetition/Presence/Frequency Penalty（慎用！）**：
- 结构化输出场景**严禁**使用惩罚项，会导致JSON格式错误
- RAG问答场景不开启Presence Penalty，会导致模型脱离检索内容编造

### 4. Prompt工程核心分类

| 类型 | 核心作用 | 场景 |
|-----|---------|-----|
| 角色Prompt | 设定固定身份约束输出 | 客服、专家答疑 |
| 少样本Prompt | 提供示例让模型模仿 | 结构化输出、复杂规则 |
| 思维链Prompt(CoT) | 分步思考再给答案 | 数学、逻辑推理 |
| RAG专属Prompt | 拼装检索知识+幻觉约束 | 知识库问答 |

### 5. 结构化输出分层方案

| 层级 | 方案 | 特点 |
|-----|------|-----|
| L1 | Prompt约束 | 零成本，合规率低 |
| L2 | 原生API约束 | 企业级首选，99%合规 |
| L3 | 自定义解码约束 | 100%合规，需开发成本 |
| L4 | 兜底修复 | 全场景兼容，合规率拉满 |

### 6. Embedding与RAG全链路

**Embedding核心特性**：
- 固定维度向量化（1024/768维）
- 语义相近→向量距离近
- 常用余弦相似度计算

**RAG全链路**：
```
离线预处理：文档清洗→分块(200-500token)→向量化→入库
在线推理：Query向量化→向量检索→重排→Prompt拼装→生成→结构化输出
```

## 二、关键技术点

### 参数优先级排序
**Temperature > Top-p/Top-k > Max Tokens/Stop Sequences > 惩罚项**

### 场景化黄金配置
| 场景 | 配置 |
|-----|-----|
| JSON/结构化输出 | T=0~0.3, Top-p=0.9, Max Tokens预留2倍, 固定seed |
| 分析/总结/评审 | T=0.4~0.8, Top-p=0.9 |
| 创作/文案 | T=0.9~1.2, Top-p=0.95 |
| 可复现/单元测试 | T=0, 固定seed, Top-k=1 |

### 避坑红线
1. 上下文窗口 ≠ 最大生成长度
2. T=0 配合固定seed才能实现幂等
3. 结构化输出绝对不能开惩罚项
4. Max Tokens不要卡着预估长度设
5. 解码参数必须和Prompt约束配合使用
